Hostatom / Plesk

ภาพรวมการนำโปรเจกต์ขึ้น Hostatom

คู่มือสำหรับเตรียมโปรเจกต์เว็บให้พร้อม deploy ขึ้น Hostatom/Plesk ใช้ได้กับ HTML/PHP, Node.js, Next.js, React, Laravel และ WordPress

Subdomain Recommended Stack Database Environment Variables Node.js App SSL / HTTPS Local Testing
1

วางแผนก่อน deploy

ก่อนนำโปรเจกต์ขึ้น host ให้ตอบคำถามเหล่านี้ให้ครบ เพื่อไม่สับสนระหว่างไฟล์ โดเมน และฐานข้อมูล

เว็บจะอยู่ที่ไหน?

เช่น app.example.com หรือ example.com/project

โปรเจกต์เป็นชนิดไหน?

Static, PHP, Node.js, Next.js, Laravel, WordPress

ต้องใช้ฐานข้อมูลไหม?

ถ้าใช้ ต้องมี database name, user, password, host, port

หลักคิดคือ "โดเมนชี้ไปที่โฟลเดอร์" — "runtime รันไฟล์เริ่มต้น" — "แอปอ่าน config จาก env" ถ้าสามอย่างนี้ตรงกัน ระบบจะ debug ง่ายมาก
2

ภาษาและเทคโนโลยีที่แนะนำ

Hostatom/Plesk รองรับเว็บหลายแบบ โดยเฉพาะ PHP และ Node.js ถ้าเริ่มโปรเจกต์ใหม่และต้องการทำเว็บแอปสมัยใหม่ ตัวเลือกยอดนิยมคือ Next.js + TypeScript + Tailwind CSS

แนะนำที่สุดสำหรับเว็บแอปใหม่

Next.js + TypeScript + Tailwind CSS

เว็บทั่วไป / ระบบหลังบ้าน

PHP, Laravel, WordPress

เว็บหน้าเดียว / static

React + Vite, HTML / CSS / JS

Stack เหมาะกับงาน สิ่งที่ต้องเตรียมบน Hostatom
Next.js + TypeScript + Tailwind เว็บแอป, dashboard, ระบบสมาชิก, API, ระบบที่ต้องขยายต่อ เปิด Node.js App, build โปรเจกต์, ตั้ง startup file, ตั้ง .env
React + Vite + TypeScript เว็บหน้าเดียว, admin panel ที่เรียก API จากที่อื่น รัน npm run build แล้วอัปโหลด dist/
Laravel ระบบ CRUD, backend, งานองค์กร ตั้ง PHP version, Composer dependencies, document root → public/
WordPress เว็บโรงเรียน, ข่าวสาร, บทความ ติดตั้งผ่าน WordPress Toolkit หรืออัปโหลดไฟล์แล้วเชื่อมฐาน
PHP ธรรมดา เว็บเล็ก, ฟอร์มง่าย ๆ, deploy ง่ายที่สุด อัปโหลดไฟล์ PHP และตั้งค่า database/config
ถ้าทำระบบใหม่ที่มี login, dashboard, ตารางข้อมูล และต้องแก้ต่อในอนาคต เลือก Next.js + TypeScript + Tailwind CSS เป็นค่าเริ่มต้นได้เลย
ก่อนเลือก Node.js/Next.js ให้ตรวจใน Hostatom ว่ามีเมนู Node.js App, เลือก Node.js version ได้ และ restart app ได้จาก Plesk
3

เตรียม Domain หรือ Subdomain

1

สร้าง subdomain ใน Hostatom/Plesk

ถ้าโปรเจกต์แยกจากเว็บหลัก ให้สร้าง subdomain ใหม่ เช่น:

project.example.com
admin.example.com
checkin.example.com
2

จำชื่อโฟลเดอร์ที่ Hostatom สร้างให้

Plesk จะสร้างโฟลเดอร์ชื่อตาม subdomain เช่น project.example.com/ — โฟลเดอร์นี้คือจุดที่จะอัปโหลดไฟล์

3

เปิด SSL

หลังสร้าง subdomain ให้ไปที่ SSL / Let's Encrypt แล้วออก certificate ให้โดเมนนั้น

ถ้าเห็นหน้า "Domain Default page" ของ Plesk แปลว่าโดเมนชี้ถูกแล้ว แต่ยังไม่ได้อัปโหลดโปรเจกต์ หรือมีไฟล์ index.html default บังอยู่
4

เตรียมฐานข้อมูล

ข้ามส่วนนี้ได้ถ้าโปรเจกต์ไม่มีฐานข้อมูล สำหรับ Hostatom/Plesk ตัวเลือกที่เหมาะที่สุดคือ MariaDB/MySQL

ประเภทฐานข้อมูล เหมาะกับงาน คำแนะนำบน Hostatom
MariaDB / MySQL ระบบทั่วไป, login, dashboard, CRUD, WordPress, Laravel, Next.js แนะนำเป็นค่าเริ่มต้นสำหรับโปรเจกต์ส่วนใหญ่
PostgreSQL query ซับซ้อน, data integrity สูง ใช้ได้ถ้า hosting มีบริการให้ หรือใช้ cloud database ภายนอก
SQLite prototype, demo, app ที่มีผู้ใช้น้อย ไม่แนะนำสำหรับ production ที่มีหลายผู้ใช้
MongoDB / NoSQL ข้อมูลเอกสาร, schema เปลี่ยนบ่อย มักต้องใช้บริการภายนอก เช่น MongoDB Atlas

หลักออกแบบ schema

  • ตั้งชื่อตารางเป็นภาษาอังกฤษ สั้น ชัดเจน เช่น users, roles
  • ทุกตารางหลักควรมี primary key เช่น id auto increment หรือ UUID
  • เก็บเวลามาตรฐานด้วย created_at และ updated_at
  • ใช้ foreign key หรืออย่างน้อยออกแบบความสัมพันธ์ให้ชัด
  • ทำ index ให้คอลัมน์ที่ใช้ค้นหาบ่อย เช่น email, status
  • รหัสผ่านต้องเก็บเป็น hash เท่านั้น ห้ามเก็บ plaintext
  • แยก master data ออกจาก transaction data
  • เตรียมไฟล์ schema.sql หรือ migration ไว้เสมอ

ตัวอย่างโครงสร้างตารางพื้นฐาน

users       → id, name, email, password_hash, role_id, created_at, updated_at
roles       → id, name
settings    → id, setting_key, setting_value
activity_logs → id, user_id, action, created_at
1

สร้าง database และ user

รายการตัวอย่าง

Passwordเก็บในที่ปลอดภัย ห้าม commit ลง git

นำ schema/data เข้า database

ใช้ phpMyAdmin, Adminer, migration command หรือ SQL import ตาม framework ที่ใช้

หลัง import ให้ตรวจจำนวน table และข้อมูลเริ่มต้น เช่น admin user, config, master data ที่ระบบต้องใช้
5

เตรียม Environment Variables

แอปจริงไม่ควร hardcode password, token หรือ connection string ในโค้ด ให้เก็บในไฟล์ .env หรือช่อง Environment Variables ของ Hostatom

ตัวอย่างสำหรับ Node.js / Next.js

DATABASE_URL=mysql://project_user:DB_PASSWORD@localhost:3306/project_db
JWT_SECRET=your-long-random-secret
NODE_ENV=production

หลักการเลือก host ฐานข้อมูล

แอปรันที่ไหน Database host ที่มักใช้
รันบน Hostatom และฐานอยู่ Hostatom เดียวกัน localhost:3306
รันในเครื่อง local แต่ต่อฐานจริงบน host IP หรือ hostname ที่ Hostatom อนุญาตให้ต่อจากภายนอก
รันกับฐานภายนอก เช่น cloud database connection string ของบริการนั้น
ห้ามอัปโหลด .env.local, password จริง, token จริง หรือ private key ขึ้น GitHub
6

ทดสอบบนเครื่อง Local ก่อน Deploy

ก่อนอัปโหลดขึ้น Hostatom ควรรันโปรเจกต์บนเครื่องตัวเองให้ผ่านก่อน โดยเฉพาะ Node.js/Next.js เพราะปัญหาเรื่อง dependency, Node version และ cache จะเจอได้ตั้งแต่ขั้นนี้

ตรวจ Node.js version

ให้ใช้เวอร์ชันเดียวกับที่ระบุใน package.json หรือที่ Hostatom รองรับ เช่น 20.x หรือ 22.x

node -v
npm -v
ถ้าโปรเจกต์ระบุ "engines": { "node": "22.x" } แต่เครื่อง local ใช้ 24.x อาจเจอ error แปลก ๆ ตอน runtime ให้สลับมาใช้ Node 22

ติดตั้ง dependency และรัน dev

npm install
npm run dev

เปิดเว็บตาม URL ที่ terminal แสดง เช่น http://localhost:3000

ทดสอบ build ก่อน deploy

npm run build

ถ้า build ไม่ผ่าน ให้แก้ให้เรียบร้อยก่อนอัปโหลด เพราะ server debug ยากกว่าเครื่อง local มาก

แก้ปัญหา: Cannot read properties of undefined (reading 'call')

มักเกิดจาก cache/chunk ของ Next.js หรือ Webpack เพี้ยน, dev server เก่าค้าง หรือ Node version ไม่ตรง

# 1) หยุด dev server ก่อน (Ctrl + C)

# 2) ล้าง cache
rm -rf .next

# 3) รันใหม่
npm run dev

จากนั้น hard refresh browser:

Mac: Cmd + Shift + R    |    Windows: Ctrl + Shift + R
ถ้ายังไม่หาย ลองเปิด Incognito/Private Window หรือเปลี่ยน port ชั่วคราว: npm run dev -- -p 3001
7

เตรียม Build จากเครื่อง

ขั้นตอนนี้ขึ้นกับชนิดโปรเจกต์

ชนิดโปรเจกต์ สิ่งที่ต้องเตรียม
HTML/CSS/JS static อัปโหลดไฟล์ตรงได้ เช่น index.html, assets/
PHP อัปโหลดไฟล์ PHP และตั้งค่า database/config
React / Vite static build รัน npm run build แล้วอัปโหลด dist/
Next.js / Node.js ต้องมี runtime Node.js, .next/, node_modules/, startup file

ตัวอย่างคำสั่ง Node.js / Next.js

npm install
npm run build
ถ้า Hostatom มี Terminal หรือปุ่ม NPM install ให้ติดตั้ง dependency บน server ได้ แต่ถ้าไม่มี ต้องอัปโหลด node_modules/ จากเครื่อง local ขึ้นไปด้วย

อัปโหลดไฟล์ขึ้น Hostatom

1

เข้า FTP

ใช้ FTP account ของ Hostatom แล้วเข้าโฟลเดอร์ subdomain หรือ document root ที่ต้องการ

2

อัปโหลดไฟล์ตามชนิดโปรเจกต์

สำหรับ Next.js/Node.js ที่ build ในเครื่อง ควรมีอย่างน้อย:

.next/
node_modules/
public/
app.js หรือ server.js
package.json
package-lock.json
next.config.mjs
.env
3

ลบไฟล์ default ที่ไม่ใช้

ถ้า Hostatom สร้าง index.html default มาให้ และโปรเจกต์เป็น Node.js/Next.js ให้ลบออก เพราะอาจบังหน้าเว็บจริง

9

ตั้งค่า Runtime

สำหรับ Node.js App

ช่องใน Hostatom/Plesk ความหมาย ตัวอย่าง
Application root โฟลเดอร์ที่มีไฟล์แอป project.example.com
Application startup file ไฟล์ที่ Passenger ใช้เริ่มแอป app.js หรือ server.js
Node.js version เวอร์ชัน Node ที่แอปรองรับ 20.x หรือ 22.x
Application mode โหมดการรัน production

ตัวอย่าง startup file สำหรับ Next.js

const next = require('next');
const { createServer } = require('http');

const port = process.env.PORT || 3000;
const app = next({ dev: false });
const handle = app.getRequestHandler();

app.prepare().then(() => {
  createServer((req, res) => handle(req, res)).listen(port);
});

11

ปัญหาที่พบบ่อย

อาการ สาเหตุที่พบบ่อย แนวทางแก้
เห็นหน้า Domain Default page ยังมีไฟล์ default หรือ runtime ยังไม่ได้ผูกกับ subdomain ลบ index.html default และตรวจ Application root
500 Web application could not be started Passenger start app ไม่ได้ ตรวจ startup file, Node version, node_modules, build output และ log
API ต่อ database ไม่ได้ .env ผิด, database user ไม่มีสิทธิ์, host/port ผิด ตรวจ connection string และลอง query database โดยตรง
CSS/JS หาย อัปโหลด build assets ไม่ครบ อัปโหลด dist/, .next/static หรือ public/ ใหม่
Cannot read properties of undefined (reading 'call') Next.js/Webpack cache เพี้ยน, browser cache เก่า, Node version ไม่ตรง หยุด dev server, ลบ .next, รัน npm run dev ใหม่ และ hard refresh
เว็บเปิด HTTP ได้ แต่ HTTPS ไม่ได้ ยังไม่ได้ออก SSL หรือ certificate ไม่ตรง subdomain ออก Let's Encrypt ใหม่ให้ subdomain

ไฟล์ log ที่ควรดู

  • logs/DOMAIN/error_log
  • logs/DOMAIN/proxy_error_log
  • tmp/boot-error.log ถ้าแอปมีระบบเขียน boot log เอง
12

Checklist ก่อนส่งมอบ

  • Domain/subdomain เปิดผ่าน HTTPS ได้
  • หน้าแรกเป็นหน้าโปรเจกต์จริง ไม่ใช่หน้า default ของ Plesk
  • API สำคัญตอบ 200
  • ระบบ login หรือ workflow หลักใช้งานได้
  • ฐานข้อมูลมี table และ seed data ที่จำเป็น
  • ไฟล์ .env บน server ถูกต้อง
  • ไม่มีไฟล์รหัสผ่านจริงถูก commit ขึ้น GitHub
  • มี README หรือคู่มือ deploy สำหรับรอบต่อไป
  • สำรองฐานข้อมูลก่อนเปิดใช้งานจริงหรือก่อนลบฐานเก่า
คู่มือภาพรวมสำหรับนำโปรเจกต์เว็บขึ้น Hostatom/Plesk
